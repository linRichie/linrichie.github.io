# Iptable 防火墙

# 什么是防火墙

防火墙：就是用于实现linux下访问控制的功能的，分为硬件和软件防火墙。防火墙工作的地方一般是在网络的边缘。任务就是：定义防火墙的策略，规则，以达到让它对出入网路的IP，数据进行检测（防火墙怎样去工作）

3.4层的防火墙，叫网络层防火墙
7层的防火墙，叫代理层的网关
三层的防火墙(网络层的防火墙)会在这层==对源地址和目标地址进行检测==。 七层的防火墙，不管源端口，目标端口，源地址，目标地址，都会对所有的东西进行检查。
就设计原理将，七层防火墙更加安全， 但却造成效率最低。一般的防火墙方案，都是两者结合。
由于：都需要从防火墙所控制的这个口进行访问，所以防火墙的工作效率就成了用户能够访问数据多少的一个最重要的控制，不好的配置容易造成流量瓶颈。

# iptables的工作原理

iptables的前身 ipfirewall iptables：能够将规则组成一个列表，实现绝对详细的访问控制功能

iptables是工作在用户控件中，定义规则的，定义的规则，可以让在内核空间当中的netfilter进行读取。而放入内核的地方必须要是特定的位置，必须是tcp/ip的协议栈经过的地方。
这个地方叫：netfilter(网络过滤器)
系统在内核空间中选择了5个位置：

  • 内核空间中：从一个网络接口进来，到另一个网络接口出去的
  • 数据包从内核流入用户空间的
  • 数据包从用户空间流出的
  • 进入/离开本机的外网接口
  • 进入/离开本机的内网接口

==iptables的工作机制== 当在做NAT和DNAT的时候，目标地址转换必须在路由之前转换。需要在外网后内网前进行设置关卡 五个位置，也成为了五个规则链（五链）

  • PREROUTING （路由前）
  • INPUT （数据包流入口）
  • FORWARD （转发网卡）
  • OUTPUT （数据包出口）
  • POSTROUTING （路由后） 上面就是netfilter（网络过滤器）的五链，只要经过本机，必将经过这5个链中的其中一个链

# 四表

==防火墙的策略==
两种：“通”策略，“赌”策略。
通策略，默认门是关着的，必须要定义谁能进。堵策略则是，大门是洞开的，但是你必须有身份认证，否则不能进。所以我们要定义，让进来的进来，让出去的出去，所以通，是要全通，而堵，则是要选择

四表：raw表，mangle表，nat表，filterbiao
五链：PREROUTING，INPUT，FORWARD，OUTPUT，POSTROUTING。

# 规则表

表链作用
filterINPUT，FORWARD，OUTPUT过滤数据包。内科模块：iptables_filter
NatPREROUTING，POSTROUTING，OUTPUT用于网络地址转换(IP,端口)。内科模块：iptables_nat
ManglePREROUTING，INPUT，FORWARD，OUTPUT，POSTROUTING修改数据包的服务类型，TTL，并且可以配置路由实现QOS。内核模块：iptables_mangle(这个表看着比较麻烦。但是设置策略时几乎都不会用到它)
RawOUOTPUT，PREROUTING决定数据包是否被状态跟踪机制处理。内核模块：iptabls_raw

# 规则链

链作用
INPUT进来的数据包应用此规则链中的策略
OUTPUT外出的数据包应用此规则链中的策略
FORWARD转发数据包时应用此规则链中的策略
PREROUTING对数据包做路由选择前应用此链中的规则（所有的书记包进来的时候都会由这个链处理）
POSTROUTING对数据包做路由选择后应用此链中的规则（所有的数据包出来的时候都会由这个链处理）

# 规则表之间的有限顺序

Raw-mangle-nat-filter
规则链之间的优先顺序（三种情况）：

# 第一种情况：入站数据流向

从外界到达防火墙的数据包，先被PREROUTING规则链处理(是否修改数据包地址等)，之后会进行路由选择(判断数据包应该发往何处)，如果数据包的目标主机是防火墙本机(比如说Internet用户访问防火墙主机中的web服务器的数据包)，那么内核将其传给INPUT链进行处理(决定是否允许通过等)，通过以后再交给系统上层的应用程序(比如apache服务器)进行响应

# 第二种情况：转发数据流向

来自外界的数据包达到防火墙后，首先被PREROUTING规则链处理，之后会进行路由选择，如果数据包的目标地址是其他外部地址(比如局域网用户通过网关访问QQ站点的数据包)，则内核将其传递给FORWARD链进行处理(是否转发或拦截)，然后交给POSTROUTING规则链(是否修改数据包的地址等)进行处理。

# 第三种情况 出站数据流向

防火墙本机向外部地址发送的数据包(比如在防火墙主机中测试公网DNS服务器时)，首先被OUTPUT规则链处理，之后进行路由选择，然后传递给POSTROUTING规则链(是否修改数据包的地址等)进行处理

# 常用命令

# 防止DDOS攻击脚本  
iptables -A INPUT -i eth0 -p tcp --syn -m connlimit --connlimit-above 15 -j DROP
iptables -A INPUT -p tcp -m state --state ESTABLISHED,RELATED -j ACCEPT

# 防止cc攻击
## 1)控制单个IP的最大并发连接数
iptables -I INPUT -p tcp --dport 80 -m connlimit --connlimit-above 50 -j REJECT
## 2)控制单个IP的某段时间的连接数
iptables -A INPUT -p icmp -m limit --limit 1/m --limit-burst 10 -j REJECT
iptables -A INPUT -p icmp -J DROP

# 拒绝所有的IP访问所有的端口
iptables -A INPUT -j REJECT --reject-with icmp-port-unreachable

# 允许公司IP访问MySQL
iptables -A INPUT -s 61.149.47.34/32 -p tcp -m tcp --dport 3306 -j ACCEPT

# 允许公司IP访问Redis
iptables -A INPUT -s 61.149.47.34/32 -p tcp -m tcp --dport 6379 -j ACCEPT

# 允许所有IP访问网站
iptables -A INPUT -p tcp -m tcp --dport 80 -j ACCEPT

#允许服务器访问外网
iptable -A INPUT -m state --state RELATED,ESTABLISHED -j ACCEPT

# 允许本地回环
iptables -A INPUT -i lo -j ACCEPT

# 允许ping请求
iptables -A INPUT -p icmp -m icmp --icmp-type 8 -j ACCEPT

# 允许所有IP访问8081 10050 65008端口
iptables -A INPUT -p tcp -m multiport --dports 8081,10050,65008 -j ACCEPT
